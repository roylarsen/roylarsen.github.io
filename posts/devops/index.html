mhqa

I’m currently working on authoring a tool in python, and that’s been fun to work on. It’s currently pretty limited in scope, but soon I should be able to start building things out and really start doing some fun Continuous Deployment/Delivery stuff for our QA people.

Long story short, this tool uses the saltstack API (probably something I’ll write a post about in the future) to interact with the vSphere API to revert back to “clean” snapshots and reinstall our backend software with a named version on the CLI. In the future, I’ll probably kill off the snapshot part of the tool (because you’re not really supposed to use snapshots like that) and just do fresh VMs for installs (and upgrade installs).

It’ll be exciting to see this tool when I finish it. It’s the first real bit of programming I’ve ever done at work (I think I’m up to ~200 lines of code, probably closer to 300 with the salt runner I wrote to revert snapshots).
